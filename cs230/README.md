This folder contains my completed Software Design Document for The Gaming Room (Draw It or Lose It).

## Client + requirements
The client was The Gaming Room. They wanted to expand Draw It or Lose It from an Android-only app to other operating platforms. The main requirements were supporting multiple games at once, keeping game/team/player names unique, and preventing duplicate game instances.

## What I did well
I kept the document organized and stayed focused on the requirements instead of adding extra stuff.

## What was helpful about the design document
Writing the design first helped me plan the classes/relationships and made the coding part more straightforward.

## One thing I would revise
I would add more detail about storage and memory choices for a real deployment (what stays in memory vs what should be persisted).

## Interpreting user needs
I turned the user needs into specific design rules. 
That matters because the design has to match what the client actually needs, not just what seems cool to build.

## My design approach
I start from requirements, map out the main objects, and keep the design simple and maintainable. 
I’d use the same approach again for a similar project.
