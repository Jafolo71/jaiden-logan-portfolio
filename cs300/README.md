This folder contains my Project One and Project Two work from CS 300, focused on data structures and algorithms.

## What problem I was solving
The main problem in this course was figuring out the best way to store and organize course data for an advising program.  
The program needed to load course data from a file, print the full course list in order, and let the user look up one course and its prerequisites.

## How I approached it / why data structures mattered
I approached it by comparing different data structures instead of jumping straight into code.  
Looking at a vector, hash table, and binary search tree helped me see how the structure you choose affects searching, sorting, and overall efficiency.

## Roadblocks and how I worked through them
The biggest roadblocks were understanding how each structure really worked and then getting the code to behave the way I wanted.  
I got through that by breaking things into smaller pieces, testing one part at a time, and using my pseudocode to stay on track.

## How this changed the way I design programs
This work made me spend more time thinking through the structure of a program before writing the code.  
It showed me that planning first usually makes the coding part go smoother and makes it easier to solve problems as they come up.

## Maintainable, readable, and adaptable code
This course pushed me to write code that was more organized and easier to follow.  
I paid more attention to function structure, naming, and comments, and I think my code is more readable now than it was before.
